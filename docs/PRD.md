# PRD：讀書筆記本系統

## 1. 專案概述

### 背景與動機
學生在閱讀大量書籍、文獻或專案參考資料時，常面臨筆記散落、難以回顧與整理的問題。為了提升學習與研究效率，需要一個能集中管理閱讀紀錄、心得及評價的「實用筆記本」系統，幫助他們將零散的知識集中與結構化。

### 目標用戶
需要做筆記的學生（包含高中生、大學生及研究生等）。

### 核心價值主張
提供一個簡單、直觀且高效率的數位讀書筆記本，讓學生能快速記錄閱讀摘要、評分，並能透過搜尋功能迅速找到過去的學習軌跡，進而提升知識吸收與應用能力。

---

## 2. 功能需求

以下列出本系統的主要功能與對應的使用者故事：

1. **記錄書名（與基本資訊）**
   - 作為 **學生**，我希望 **能新增一筆閱讀紀錄並填寫書名等資訊**，以便 **我能清楚追蹤自己讀過哪些書**。
2. **撰寫心得**
   - 作為 **學生**，我希望 **能在每本書的紀錄中撰寫詳細的閱讀心得與重點筆記**，以便 **我未來回顧時能快速找回記憶與靈感**。
3. **書籍評分**
   - 作為 **學生**，我希望 **能對讀完的書籍給予相對應的評分（如1到5星）**，以便 **我能快速篩選出對我最有價值的讀物**。
4. **內容搜尋**
   - 作為 **學生**，我希望 **能透過搜尋列查找書名或心得內容**，以便 **在需要時能立刻找到相關的過去筆記**。
5. **標籤與分類（擴充功能）**
   - 作為 **學生**，我希望 **能為筆記加上不同的標籤（如：文學、歷史、工具書）**，以便 **我更有系統地分類管理大量的筆記資料**。

---

## 3. 非功能需求

### 技術限制
- **框架與語言**：使用 Python 搭配 Flask 輕量級網頁框架進行開發。
- **前端介面**：使用 Jinja2 模板引擎結合 HTML/CSS 呈現畫面。
- **資料儲存**：使用 SQLite 作為輕量級關聯式資料庫來儲存使用者的筆記資料。

### 效能與安全考量
- **效能**：頁面載入與搜尋結果的呈現應在 2 秒以內完成。
- **安全**：需對使用者的輸入層包含防護，避免 XSS 攻擊或 SQL Injection 等安全性漏洞；保護使用者的筆記資料完整性。

---

## 4. MVP 範圍

### Must Have (必須擁有)
- 新增並儲存一筆完整的資料（書名、心得、評分）。
- 所有讀書筆記的列表檢視。
- 透過關鍵字搜尋特定筆記。

### Should Have (應該擁有)
- 修改（編輯）既有的筆記內容。
- 刪除不需要的筆記。

### Nice to Have (可以擁有)
- 提供標籤（Tag）系統，允許依照標籤篩選。
- 基本的閱讀數量或評分統計圖表。
- 匯出筆記內容（例如 Markdown 或純文本格式）的功能。
